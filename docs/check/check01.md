

# Phase 1 富文本编辑器：验收标准

## 核心验收原则

```
一句话定义Phase 1的交付目标：

"一个作家拿到它，在没有任何AI功能的情况下，
 愿意用它来写小说的前三章。"

不是"能用"，而是"愿意用"。
这两个标准之间的差距，就是Phase 1要填的全部内容。
```

## 1. 输入与编辑体验

### 1.1 基础输入

```
验收测试：

☐ T-1.1.1  打开编辑器，直接开始打字，文字即时出现
           → 不需要点击任何按钮或选择任何选项

☐ T-1.1.2  连续打字 2000 字，全程无卡顿、无丢字、无延迟
           → 主观感受：和在记事本里打字一样流畅

☐ T-1.1.3  快速打字测试：以最快速度连续输入 30 秒
           → 每个字都正确出现，顺序正确，无吞字

☐ T-1.1.4  中文输入法兼容：
           → 搜狗拼音：正常（包括长句输入、候选词选择）
           → 微软拼音：正常
           → macOS 原生输入法：正常
           → 输入法候选窗口位置正确（不被遮挡、不飘到屏幕角落）

☐ T-1.1.5  回车键创建新段落
           → 新段落自动获得唯一 UUID
           → 光标移动到新段落开头

☐ T-1.1.6  Shift+回车创建软换行（段落内换行）
           → 不产生新段落，不生成新 UUID

☐ T-1.1.7  空行行为：
           → 连续按两次回车 → 产生一个空段落（有UUID）
           → 视觉上有明显的段落间距
```

### 1.2 选择操作

```
☐ T-1.2.1  鼠标点击定位光标 → 精确到字符间
☐ T-1.2.2  鼠标拖拽选择文字 → 蓝色高亮
☐ T-1.2.3  双击选中一个词（中文：一个词组 / 英文：一个单词）
☐ T-1.2.4  三击选中整个段落
☐ T-1.2.5  Ctrl+A 全选
☐ T-1.2.6  Shift+方向键 扩展/收缩选区
☐ T-1.2.7  Ctrl+Shift+左/右 按词选择
☐ T-1.2.8  选区跨段落：从第一段拖到第三段，三个段落的内容都被选中
```

### 1.3 剪切/复制/粘贴

```
☐ T-1.3.1  Ctrl+C 复制选中文字，Ctrl+V 粘贴
           → 在编辑器内部：保留格式（粗体等）
           → 粘贴的段落获得新 UUID（不复制原 UUID）

☐ T-1.3.2  Ctrl+X 剪切
           → 剪切的内容可粘贴到其他位置
           → 剪切后原位置的段落 UUID 正确处理

☐ T-1.3.3  从外部粘贴纯文本（如记事本）
           → 正确插入，自动分段
           → 每段获得新 UUID
           → 无格式残留

☐ T-1.3.4  从外部粘贴富文本（如 Word、网页）
           → 剥离所有外部样式
           → 只保留基础格式（粗体、斜体）
           → 不带入任何 CSS、字体、颜色信息
           → 纯净 JSON 不被污染

☐ T-1.3.5  从浏览器粘贴一段网页内容
           → 验证粘贴后保存的 JSON 中无 style 属性
           → 在 Console 中调用 getContent('json')
           → 检查输出中不含任何 CSS 相关字段
```

### 1.4 撤销/重做

```
☐ T-1.4.1  Ctrl+Z 撤销上一步操作
☐ T-1.4.2  Ctrl+Shift+Z（或 Ctrl+Y）重做
☐ T-1.4.3  撤销深度 ≥ 100 步
☐ T-1.4.4  撤销/重做后光标位置正确
☐ T-1.4.5  撤销删除段落 → 段落恢复，UUID 与被删除前一致
☐ T-1.4.6  连续撤销多步 → 每一步的中间状态都正确
```

## 2. 格式化能力

### 2.1 小说写作必需的格式

```
☐ T-2.1.1  粗体（Ctrl+B）
           → 选中文字 → Ctrl+B → 文字加粗
           → 再次 Ctrl+B → 取消加粗
           → 保存后重新打开 → 粗体还在

☐ T-2.1.2  斜体（Ctrl+I）
           → 同上测试流程

☐ T-2.1.3  标题层级
           → H1：章节标题（如"第一章 相遇"）
           → H2：场景/节标题（如"一"、"上午"）
           → H3：可选的更细分节
           → 快捷键：Ctrl+1/2/3 或 通过工具栏
           → 标题视觉上有明确的大小层级区分

☐ T-2.1.4  引用块
           → 用于小说中的信件、诗句、回忆片段
           → 视觉上有明确的缩进和左边框
           → 快捷键或 Slash Command

☐ T-2.1.5  分割线（水平线）
           → 用于场景切换
           → 输入 --- 或 *** 后回车自动转换
           → 或通过 Slash Command 插入

☐ T-2.1.6  删除线（可选但推荐）
           → 用于标记"考虑删除"的段落
```

### 2.2 格式边界测试

```
☐ T-2.2.1  一个段落中同时存在粗体和斜体
           → 保存 → 重新打开 → 格式保持正确

☐ T-2.2.2  粗斜体（同时加粗和斜体）
           → 保存 → 重新打开 → 格式保持正确

☐ T-2.2.3  标题段落中包含粗体/斜体
           → 正确渲染

☐ T-2.2.4  连续切换格式 10 次
           → 格式状态一直正确，不混乱

☐ T-2.2.5  格式工具栏状态同步
           → 光标移入粗体文字 → 工具栏粗体按钮高亮
           → 光标移到普通文字 → 工具栏粗体按钮取消高亮
```

### 2.3 Slash Command

```
☐ T-2.3.1  在空段落输入 "/" → 弹出命令面板
☐ T-2.3.2  命令列表至少包含：
           → /h1  /h2  /h3  → 设置标题
           → /quote          → 插入引用块
           → /divider        → 插入分割线
           → /text           → 恢复为正文
☐ T-2.3.3  输入 "/h" → 列表过滤，只显示 h1/h2/h3
☐ T-2.3.4  上下键选择 → 回车确认
☐ T-2.3.5  Esc 关闭命令面板
☐ T-2.3.6  在非空段落行首输入 "/" → 同样弹出
☐ T-2.3.7  在段落中间输入 "/" → 不弹出（避免误触）
```

## 3. 查找与替换

```
☐ T-3.1  Ctrl+F 打开查找栏
         → 出现在编辑器顶部或底部
         → 光标自动聚焦到搜索输入框

☐ T-3.2  输入搜索词 → 文档中所有匹配项高亮标记
         → 当前匹配项有特殊颜色（如橙色背景）
         → 其他匹配项有普通颜色（如黄色背景）
         → 显示匹配数量（如"3/12"表示当前第3个，共12个）

☐ T-3.3  Enter 或 ↓ 按钮 → 跳到下一个匹配
         → 视图自动滚动到该位置
         → Shift+Enter 或 ↑ 按钮 → 跳到上一个匹配

☐ T-3.4  Ctrl+H 打开替换模式
         → 在查找栏下方出现替换输入框
         → [替换] 按钮：替换当前匹配项，并跳到下一个
         → [全部替换] 按钮：一次性替换所有匹配项
         → 全部替换后显示"已替换 N 处"

☐ T-3.5  大小写敏感开关
         → 默认不敏感
         → 点击切换后搜索结果实时更新

☐ T-3.6  正则表达式开关（可选但推荐）
         → 开启后支持正则搜索
         → 如搜索 "他[说道]+" 能匹配"他说""他道""他说道"

☐ T-3.7  Esc 关闭查找栏 → 光标回到之前的位置
         → 高亮标记消失

☐ T-3.8  搜索空文档 → 不崩溃，显示"无匹配"
☐ T-3.9  搜索无结果的关键词 → 不崩溃，显示"0/0"

☐ T-3.10 全部替换后可整体撤销（Ctrl+Z 一次撤销全部替换）
```

## 4. 文件管理（单文件阶段）

> Phase 1 先做单文件管理（一次编辑一个文件）。多文件/项目系统是 Phase 2。

### 4.1 保存

```
☐ T-4.1.1  Ctrl+S 手动保存
           → 标题栏的修改标记消失（如 * 号消失）
           → 底部或标题栏显示"已保存"提示
           → 保存到磁盘的时间 < 100ms（用户无感知延迟）

☐ T-4.1.2  自动保存
           → 停止输入 N 秒后（建议30秒）自动保存
           → 编辑器失去焦点（切换窗口）时自动保存
           → 自动保存不打断用户写作（无弹窗、无卡顿）
           → 自动保存后标题栏标记更新

☐ T-4.1.3  修改标记
           → 有未保存修改时标题栏显示 "•" 或 "*"
           → 保存后标记消失
           → 新的修改产生后标记重新出现

☐ T-4.1.4  保存产物验证（在文件管理器中检查）：
           → 存在 .tiptap.json 文件（编辑器格式）
           → 存在 .md 文件（Markdown 格式）
           → JSON 文件内容纯净，无 CSS（手动打开检查）
           → Markdown 文件可读（手动打开检查）
           → 两者内容一致（文字内容相同）
```

### 4.2 打开

```
☐ T-4.2.1  启动应用 → 如果上次有打开的文件 → 自动恢复
           → 内容完整
           → 格式完整
           → 光标位置恢复到上次关闭时的位置（可选但推荐）

☐ T-4.2.2  菜单/按钮 → 打开文件 → 文件选择对话框
           → 选择 .tiptap.json 文件 → 编辑器加载内容
           → 格式正确渲染

☐ T-4.2.3  打开时如果当前有未保存修改 → 弹出提示
           → "当前文件有未保存的修改，是否保存？"
           → [保存] [不保存] [取消]
```

### 4.3 新建

```
☐ T-4.3.1  新建文件 → 编辑器清空 → 可以开始写作
☐ T-4.3.2  新建时如果当前有未保存修改 → 同上提示
☐ T-4.3.3  新建文件第一次保存 → 弹出"另存为"对话框选择位置
```

### 4.4 关闭保护

```
☐ T-4.4.1  关闭窗口时有未保存修改 → 弹出确认
           → "文件有未保存的修改，是否保存后退出？"
           → [保存并退出] [不保存退出] [取消]

☐ T-4.4.2  系统强制关闭（如电脑重启）→ 自动保存的内容不丢失
           → 下次打开可恢复
```

## 5. 界面与布局

### 5.1 编辑区域设计

```
☐ T-5.1.1  编辑区域居中显示，两侧留白
           → 内容区宽度 650-800px（可配置）
           → 类似 A4 纸面的感觉
           → 两侧灰色/淡色背景

☐ T-5.1.2  行高舒适
           → 行高 1.75-2.0（适合中文长文阅读）
           → 段落间距明显大于行间距

☐ T-5.1.3  字体
           → 默认使用适合中文阅读的字体
           → 正文字号 16-18px
           → 字色不是纯黑（#333 或 #1a1a1a，减轻视觉疲劳）

☐ T-5.1.4  持续写作 30 分钟不感到视觉疲劳
           → 主观测试：让人实际写作30分钟后评估
```

### 5.2 工具栏

```
☐ T-5.2.1  顶部工具栏包含：
           ┌───────────────────────────────────────────────┐
           │  B   I   S  │  H1  H2  H3  │  "  ──  │  ⛶   │

☐ T-5.2.3  工具栏状态反映当前光标位置的格式

☐ T-5.2.4  工具栏不过度占用垂直空间（一行即可）

### 5.3 主题

```
☐ T-5.3.1  亮色主题（默认）
           → 白色/米白背景
           → 深色文字
           → 柔和的 UI 元素颜色

☐ T-5.3.2  暗色主题
           → 深色背景（不是纯黑，建议 #1a1a2e 或类似）
           → 浅色文字（不是纯白，建议 #e0e0e0）

☐ T-5.3.3  主题切换
           → 设置中切换 或 快捷键切换

☐ T-5.3.4  跟随系统主题（可选）

### 5.4 全屏/沉浸模式

```
☐ T-5.4.1  快捷键（如 F11 或 Ctrl+Shift+F）进入全屏

☐ T-5.4.2  Esc 退出全屏 → 恢复正常布局

☐ T-5.4.3  全屏模式下：

           → Ctrl+F 仍能查找（查找栏临时出现）

## 8. 已有基础设施验证 ⭐

```

☐ T-8.1.3  删除第 3 段
           → 其他段落的 UUID 不变
           → Console 验证

☐ T-8.1.4  剪切第 2 段 → 粘贴到第 4 段后面
           → 粘贴后的段落保留原来第 2 段的 UUID

           → Console 验证

☐ T-8.1.5  复制第 2 段 → 粘贴
           → 粘贴产生的新段落有新 UUID（不同于第 2 段）
           → 原第 2 段 UUID 不变

☐ T-8.1.6  撤销删除段落
           → 恢复的段落 UUID 与删除前一致
           → Console 验证

☐ T-8.1.7  选中跨越 3 个段落的文字 → 删除
           → 合并后的段落 UUID 为原第一个段落的 UUID
           → Console 验证

☐ T-8.1.8  从外部粘贴 5 段文字
           → 每段都获得全新 UUID
           → 不与已有段落 UUID 冲突

☐ T-8.1.9  保存 → 关闭 → 重新打开
           → 所有段落的 UUID 与保存前完全一致
           → Console 验证

☐ T-8.1.10 拆分段落（在段落中间按回车）
           → 前半段保留原 UUID
           → 后半段获得新 UUID
```

### 8.2 Headless API 验证

```
☐ T-8.2.1  在浏览器 Console 中执行以下操作，每个都正确：

读取操作：
  editor.api.getContent('json')      → 返回完整 JSON
  editor.api.getContent('markdown')  → 返回 Markdown 文本
  editor.api.getContent('text')      → 返回纯文本
  editor.api.getWordCount()          → 返回字数（与状态栏一致）
  editor.api.getAllParagraphIds()     → 返回所有段落 ID 数组
  editor.api.getParagraphText('某ID') → 返回该段落纯文本
  editor.api.getCursorPosition()     → 返回 { paragraphId, offset }

写入操作：
  editor.api.insertText("测试文字")
    → 在光标位置插入，编辑器实时更新，UI 可见

  editor.api.replaceText("某段落ID", "旧文字", "新文字")
    → 替换成功，返回 true
    → 编辑器实时更新

  editor.api.insertParagraphAfter("某段落ID", "新段落内容")
    → 新段落出现在指定段落后
    → 返回新段落的 UUID

  editor.api.deleteParagraph("某段落ID")
    → 段落消失
    → 其他段落不受影响

光标操作：
  editor.api.moveCursorToParagraph("某段落ID")
    → 光标跳转到该段落开头
    → 编辑器滚动到可见位置

  editor.api.moveCursorToEnd()
    → 光标跳转到文档末尾

☐ T-8.2.2  连续通过 Console 执行 10 个操作
           → 每个操作的结果都反映在编辑器 UI 上
           → 不出现数据与 UI 不一致的情况

☐ T-8.2.3  通过 Console 操作后 Ctrl+Z 可以撤销
           → API 操作和手动操作共享同一个 undo 栈
